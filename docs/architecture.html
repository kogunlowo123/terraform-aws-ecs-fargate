terraform-aws-ecs-fargate

Architecture Document — Production-Ready AWS ECS Fargate Container Platform

Executive Summary

A production-ready Terraform module for deploying containerized applications on AWS ECS Fargate with built-in support for service discovery (AWS Cloud Map), application auto scaling based on CPU utilization, deployment circuit breaker with automatic rollback, blue/green deployments via ALB target group integration, Container Insights, ECS Exec for interactive debugging, and IAM roles for both task execution (image pull, secrets) and task-level application permissions. The module manages the full stack from cluster to service with CloudWatch log integration.

Data Flow

1
Incoming traffic hits the ALB/NLB and is routed to the target group with health check validation.
2
Target group forwards requests to the ECS service, which distributes across running Fargate tasks.
3
Task execution role pulls container images from ECR and injects secrets from SSM/Secrets Manager.
4
Containers run with the task role for application-level AWS API access (S3, DynamoDB, etc.).
5
Container logs stream to CloudWatch Log Groups. Container Insights collects cluster-level metrics.
6
Auto scaling adjusts desired task count based on average CPU utilization (default target: 70%).
7
Circuit breaker monitors deployments and automatically rolls back on consecutive task failures.
8
Cloud Map registers healthy tasks for DNS-based service-to-service discovery within the VPC.

Security Controls

Separate Execution and Task Roles

Task execution role handles infrastructure concerns (image pull, secrets); task role handles application permissions. Clear separation of duties.

Security Group Least Privilege

Security group allows inbound traffic only on the configured container port. Egress is open for outbound connectivity.

Secrets Injection via SSM/Secrets Manager

Sensitive configuration is injected at runtime from SSM Parameter Store or Secrets Manager, never baked into images.

Deployment Circuit Breaker

Enabled by default with automatic rollback, preventing bad deployments from taking down the service.

Private Networking

Tasks run in private subnets by default (assign_public_ip = false), requiring NAT or VPC endpoints for outbound access.

Industry Adaptation

Healthcare

Private subnet deployment with no public IPs. Secrets injection for PHI-related credentials. CloudWatch logs with configurable retention for audit trails.

Finance

Circuit breaker prevents trading system outages from bad deployments. Auto scaling handles market-hours traffic. Service discovery for microservice mesh.

Government

Fargate removes host management burden for FedRAMP. IAM role separation satisfies NIST access control requirements. Container Insights for continuous monitoring.

Retail

Auto scaling from 1 to N tasks for seasonal demand. Blue/green deployments for zero-downtime releases. Multi-container sidecar pattern for caching proxies.

SaaS

Cloud Map for internal service discovery across microservices. ECS Exec for live debugging without SSH. CPU-based scaling for multi-tenant workloads.

Production Readiness Checklist

  • Enable deployment circuit breaker with automatic rollback
  • Configure auto scaling with appropriate min/max capacity and CPU target
  • Enable Container Insights for cluster-level metrics and anomaly detection
  • Set CloudWatch log retention to meet compliance requirements
  • Use private subnets with NAT gateway for outbound access
  • Inject secrets via SSM Parameter Store or Secrets Manager (never environment variables)
  • Configure health check path to match application readiness endpoint
  • Set appropriate task CPU and memory for the workload profile
  • Enable service discovery for internal service-to-service communication
  • Test rollback behavior by deploying a known-bad container image
